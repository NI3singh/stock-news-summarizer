# QuantMind ML Module — How It Works & How To Use It

This folder (`quantmind/ml/`) is the **ML layer**. It has **two independent
sub-systems**, and they have *very different* "what do I do" answers — which is
the usual source of confusion:

| Sub-system | What it does | Works… |
|---|---|---|
| 🧠 **Signal model** | Predicts whether a stock goes **UP or DOWN** next day, learned from your own analysis history | …only **after ~10 days** of analyses (it needs data) |
| 🕸️ **Entity graph** | Maps **companies / people / products / relationships** from the news (e.g. *Apple → supplies → TSMC*) | …**immediately** (one analysis is enough) |

---

## 🚀 TL;DR — what you actually have to do

**You don't have to "set up" anything. You just have to USE the system normally, and the ML builds itself.**

1. **Add some tickers** (one time):
   ```
   quantmind add-ticker AAPL
   quantmind add-ticker MSFT
   quantmind add-ticker NVDA
   ```

2. **Run analyses** — this is the only thing that takes time. Either:
   - **Automatic:** leave the scheduler running → it analyzes your watchlist daily:
     ```
     quantmind run-scheduler
     ```
   - **Manual:** run this **once a day**:
     ```
     quantmind analyze AAPL MSFT NVDA
     ```

3. **The entity graph works right away** (after even one analysis):
   ```
   quantmind entity-graph AAPL
   ```

4. **The prediction model needs ~10 days of analyses.** After that:
   ```
   quantmind train-all-models     # builds the models
   quantmind predict AAPL         # UP/DOWN prediction
   ```
   …and from then on, `quantmind analyze AAPL` shows an **`ML SIGNAL:`** line automatically.

> **The short version:** keep using `analyze` (or `run-scheduler`) daily. The entity
> graph is instant; the prediction model "wakes up" once it has ~10 days of history.

---

## 🧩 What this module is (and why)

The whole point: **QuantMind learns from its own output.** Every time you run an
analysis, it stores the result. The ML layer then:

1. **Learns a predictive signal** from that history — *"when sentiment looked like
   this in the past, did the stock go up the next day?"*
2. **Builds a knowledge graph** of who's connected to whom in the news.

Both are then queryable from the **CLI** and from **Claude Desktop** (via MCP tools).

---

## 🧠 Sub-system 1 — The Signal Model (needs data)

### How it learns
- Every `analyze` run saves an analysis (its **sentiment score**, etc.) to the database.
- To create a *training example*, the model pairs each past analysis with **what the
  price actually did the next trading day** (fetched from Yahoo Finance).
  → `sentiment on day N` → label: `did the stock close up on day N+1?`
- It trains a simple, interpretable **Logistic Regression** classifier per ticker.

### ⚠️ Why it needs ~10 days (the #1 confusion)
Each **day** of analysis = **one** training example. And an example can only be
labeled once the **next day's price exists**. So:
- 1 day of analyses → ~1 sample → **not enough** → the model honestly refuses to train.
- ~10+ days of analyses → ~10+ samples → it trains, cross-validates, and reports
  whether it actually **beats a coin-flip baseline** (it's honest — a weak signal
  is reported as weak).

You **cannot** shortcut this by running `analyze` 10 times today — they'd all be the
same calendar day. It genuinely needs **calendar days** to pass.

### What you'll see before vs. after
```
# Before (today, sparse data):
$ quantmind train-model AAPL
  ✗ Need at least 10 analyses to train. Currently have 1. Keep running daily analyses.

# After ~10 days:
$ quantmind train-model AAPL
  ✓ Trained on 12 samples. CV accuracy: 58.3%
  Top feature: sentiment_score (coeff: 0.4912)

$ quantmind predict AAPL
  ML Signal Prediction — AAPL
  Prediction: UP
  Confidence: 61.2%
  Signal Strength: moderate
```

### Where models are stored
Trained models are saved as `quantmind_models/<TICKER>_model.joblib` (this folder
is git-ignored — models are regenerated by training).

## 🕸️ Sub-system 2 — The Entity Graph (works now)

### How it works
- Every `analyze` run **also** asks the LLM to extract entities and **factual
  relationships** from the news (companies, people, products, events, regulators).
- These are saved to the `entities` and `entity_relationships` DB tables.
- A **NetworkX** graph is built on top for analytics (who's most connected, who
  bridges clusters, etc.).

### What you'll see
```
$ quantmind analyze AAPL          # (this populates the graph)
$ quantmind entity-graph AAPL

Entity Graph (13 nodes, 10 edges)
  Density: 0.0641  |  Types: {'company': 8, 'person': 1, 'product': 2, ...}

Most connected (degree centrality):
  Apple  (0.8333)

Key connectors (bridges):
  Apple  (0.1212)

Relationships:
  Apple --[partners_with]--> Intel
  Apple --[supplies]--> TSMC
  Tim Cook --[works_for]--> Apple
  UK tribunal --[sues]--> Apple
```
The more you analyze, the richer the graph gets. **No 10-day wait here** — it's
useful after the very first analysis.

## 🔌 Using it from Claude Desktop (MCP)

Both sub-systems are exposed as MCP tools, so you can just *ask Claude*:

1. Start the server:
   ```
   quantmind mcp-server
   ```
2. In **Claude Desktop** → Settings → Developer → MCP Servers → Add →
   URL: `http://127.0.0.1:8765/mcp`
3. Then chat naturally:
   - *"What's the QuantMind ML signal for AAPL?"* → calls `get_ml_signal`
   - *"Show me AAPL's entity relationship graph"* → calls `get_entity_graph`
   - *"Analyze NVDA"* / *"What's in my watchlist?"* → other tools

(If no model is trained yet, `get_ml_signal` honestly replies "no trained model yet.")

## 📋 Command reference

| Command | What it does | Needs data? |
|---|---|---|
| `quantmind analyze TICKER...` | Full analysis; **updates the entity graph** + attaches an ML signal if a model exists | — |
| `quantmind ml-status` | Per-ticker: how many training samples + is a model trained | — |
| `quantmind train-model TICKER` | Train one ticker's model | 10+ days |
| `quantmind train-all-models` | Train models for the whole watchlist | 10+ days |
| `quantmind predict TICKER` | UP/DOWN prediction from the trained model | trained model |
| `quantmind entity-graph [TICKER]` | Entity graph + NetworkX analytics | 1+ analysis |
| `quantmind mcp-server` | Serve ML/graph tools to Claude Desktop | — |

## ❓ FAQ / troubleshooting

**"`train-model` says insufficient_data."**
Expected — you don't have ~10 days of analyses yet. Keep running `analyze` (or
`run-scheduler`) daily. This is *not* an error; the model refuses to pretend.

**"`analyze` doesn't show an ML SIGNAL line."**
That line only appears once a model is **trained** for that ticker. No model → no line.

**"`predict` says 'No recent analysis'."**
Run `quantmind analyze TICKER` first (the prediction uses the latest analysis).

**"`entity-graph` shows 0 nodes."**
Run `quantmind analyze TICKER` at least once — extraction happens during analysis.

**"Does the model have to be accurate?"**
No. The goal is that it **exists, is measured, and is honest** — it reports its
cross-validated accuracy vs. a baseline and tells you if the edge is weak.
